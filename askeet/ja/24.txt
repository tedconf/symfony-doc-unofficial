24日目: 次は？
=============

復習
----

23日間において、私達はsymfonyとPHP5でweb 2.0アプリケーションを構築してきました。 昨日はaskeetの開発の最後の段階で、完全な国際化対応しました。 [www.askeet.com](http://www.askeet.com/)をブラウザで訪れれば、symfonyによるおよそ3日(24時間)の仕事の結果を見ることが出来ます。 ごらんの通り、アプリケーションはあなたのチョコレート、sex、天文学、PHPプログラミングについて質問の回答をする準備ができています。

しかし、それよりもaskeetはオープンソースのプロジェクトで、次に来ること、願わくは長い話になります。

使う
----

askeetウェブサイトは一般公開されています。askeetについて宣伝とあなたの友人と身内に話すことができます。いくつかのテスト投稿は除去されますが、多くの存在している質問とユーザアカウントはそのままです。askeetは回答を見つけるための素晴らしいツールです 　多くのユーザが訪問してくれたらですが。そういうわけで評判を広めましょう。

アカウントを公開することは早くて簡単で、ニックネームとパスワードとEメール以外は要求されません。質問に対する興味を明らかにすることができ、新しい質問をすることと回答を評価することができます。Eメールアドレスはいかなる広告目的にも使われることはありません。

RSSフィードを購読して、最新の質問もしくはあなたの質問に対する最新の質問の情報が通知されるようにして下さい。

Paypalアカウントにユーザのプロファイルがリンクされているのであれば、askeetはお金儲けのための方法にもなります。

全般的に、askeetを毎日使わないためのよい理由はありません。あなたがサイトをブックマークして日常的に訪問をし、回答と質問を投稿して下さるのであれば、私達はとてもうれしいです。

インストールする
----------------

askeetはウェブサイトというより、オープンソースのプロジェクトです。 今日の時点で、askeetは既にダウンロードすることができ、どこでもインストールすることができます。今日のバージョンは1.0とタグ付けされ、利用すること、改造すること、カスタマイズすること、およびサードパーティのアプリケーションに統合することは自由にできます。

技術的に可能な理由はaskeetはPHP5とsymfonyによって構築されているからであり、法律的に可能な理由はaskeetはオープンソースのプロジェクトでMITライセンスの元で公開されているからです。

### ダウンロードする

askeetをインストールするためには、二つの選択があります:

* symfony-projectのウェブサイトから[.tgz形式のアーカイブ](http://www.symfony-project.com/downloads/askeet_release_1_0.tgz)をダウンロードする
* [SVNリポジトリ](http://svn.askeet.com/tags/release_1_0/)にある1.0のリリースからあなた自身のaskeetフォルダにチェックアウトする

symfonyプロジェクトを持ちましたら、ウェブサーバを構成すると同時に動かす準備をして下さい。

>**Note**: 全ソースは[askeetのtracのサイト](http://trac.askeet.com/trac/browser/trunk/)でブラウズすること出来ます。

### ドキュメント

24のアドベントカレンダーチュートリアルは[symfony projectのウェブサイト](http://www.symfony-project.com/askeet)でオンラインからまだ利用可能です。

askeetを外国語に翻訳したいのであれば、[Markdownバージョン](http://svn.symfony-project.com/doc/branches/1.0/askeet/) のチュートリアルをsymfonyのSubversionリポジトリで見つけることができます。askeetのチュートリアル翻訳を私達のリポジトリでホストをすることを喜んでいたします。symfonyのサイトはメディアを含んだシンタックスカラーリングのフォーマットを行うMarkdownコンバータを使用しています。そして翻訳したものは同じメディアを呼び出し、UTF-8でエンコードしてMarkdownバージョンで送って下さい。そして私達はそれを公開します。

>**Note**: 翻訳する前に、二人の人間が同時に翻訳を始めないように[askeetフォーラム](http://www.symfony-project.com/forum/index.php/t/275/)に投稿して下さい。すぐに利用可能にできるようにするため、翻訳した章を一つずつコミットして下さい。

### ファイル構造

askeetプロジェクトのアーカイブを解凍したあとで、symfonyプロジェクトの古典的なファイルの木構造ディレクトリのりストを入手することができます。symfonyブックの[ファイル構造の章](http://www.symfony-project.com/book/1_0/04-The-Basics-of-Page-Creation)で詳細が解説されています。

1.0のリリースにおいて、askeet projectは一つのアプリケーション('`frontend`'と呼びます)と11のモジュールを含みます:

    modules/
      administrator
      answer
      api
      content
      feed
      mail
      moderator
      question
      sidebar
      tag
      user

### データモデル
 
askeetはMySQL、PostgreSQL、Oracle、MSSQLおよび[Creole](http://creole.phpdb.org/wiki/)のドライバーが存在するすべてのデータベースと互換性があります。1.0のリリースにおけるaskeetのアプリケーションデータモデルは次の通りです:

![ERD](/images/askeet/mcd4.gif)

`askeet/data/sql/lib.model.schema.sql`に存在するデータベースにそれらのテーブルを追加することができるSQLクエリを見つけることができます。

`askeet/data/fixtures/`にテストデータのセットがあります。データベースに投入するためにそれらを使いたいのであれば、プロジェクトのrootディレクトリから次のコマンドを行って下さい:

    $ php batch/load_data.php

...プロジェクトのrootディレクトリからです。

貢献する
--------

askeetアプリケーションは生きているオープンソースのプロジェクトです。そういうものとして、私達は改善を続けることを望みますし、そのためにあなたの助けを必要とします。

askeetはsymfonyプロジェクトの開発リーダーでもある[Fabien Potencier](mailto:fabien.potencier@symfony-project.com)によって開発されました。フレームワークが既に重要な仕事の量を象徴するようになったので、プロジェクトを活動的にするためには熱狂的なaskeetの利用者の方からの貢献が必要とされます。たくさんやることがあります!　askeetへの貢献に興味がある開発者でしたら、次のTODOリストをご覧下さい:

* 追加された機能:
  - 複数のプレゼンテーションのための代替的なサイトのデザイン。ほとんどがグラフィカルデザインとCSSコーディングの仕事です。
  - ユーザ投稿へのCaptcha。単純な質問(例えば"一つの手に指は何本ですか？"投稿に対して自動化スパムを避けるためです。
  - 大きなタイポを避けるために投稿する前の質問のプレビュー機能
  - Eメールによるユーザ投稿のお知らせ(オプション)
  - オートログイン(cookieよる)
  - モデレータのためのRSSフィードのレポート
  - Userプロファイルページでの投稿/興味のAjaxパジネーション
  - 質問のフロントページが経時変化するためのポピュラーアルゴリズム 
  - ...
* プロジェクトの強化:
  - ユニットテスト
  - PHPdoc形式のコードドキュメンテーション
  - インストールマニュアル

加えて、検知・追跡・修正するためのバグがあります。

強化とバグのために、[askeetのチケットシステム](http://trac.askeet.com/trac/report/1)を使用して下さい。[プロジェクトのタイムライン](http://trac.askeet.com/trac/timeline)を定期的に参考にしながら、すべてのaskeetの変更を追跡することができます。

すべての貢献を歓迎します。プロジェクトをよく理解している定期的な貢献者にはすぐにSVNリポジトリにコミットする権限を与えます。

最後に、askeetに関して討論をしたい場合は、symfonyプロジェクトのウェブサイトの[askeetのフォーラムセクション](http://www.symfony-project.com/forum/index.php/f/8/)か、 [trac.askeet.com/trac/wiki](http://trac.askeet.com/trac/wiki)上の専用のwikiかを選ぶことができます。
